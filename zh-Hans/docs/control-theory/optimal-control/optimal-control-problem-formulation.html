Optimal Control Problem Formulation

This document introduces the fundamental concepts of optimal control problems using the unicycle model as a practical example. We explore different control objectives from basic parking problems to advanced collision avoidance scenarios.

Unicycle Model​

The unicycle model serves as an excellent example for understanding nonlinear optimal control problems due to its simplicity and practical relevance in robotics.

State Space Representation​

The state vector is defined as:

x(t)=[x1(t)x2(t)x3(t)x4(t)]=[px(t)py(t)v(t)θ(t)]x(t) = \begin{bmatrix} x_1(t) \\ x_2(t) \\ x_3(t) \\ x_4(t) \end{bmatrix} = \begin{bmatrix} p_x(t) \\ p_y(t) \\ v(t) \\ \theta(t) \end{bmatrix}x(t)=​x1​(t)x2​(t)x3​(t)x4​(t)​​=​px​(t)py​(t)v(t)θ(t)​​

Where:

  • px(t)p_x(t)px​(t): position in x direction
  • py(t)p_y(t)py​(t): position in y direction
  • v(t)v(t)v(t): linear velocity
  • θ(t)\theta(t)θ(t): wheel orientation angle

Control Input​

The control input vector is:

u(t)=[u1(t)u2(t)]=[α(t)ω(t)]u(t) = \begin{bmatrix} u_1(t) \\ u_2(t) \end{bmatrix} = \begin{bmatrix} \alpha(t) \\ \omega(t) \end{bmatrix}u(t)=[u1​(t)u2​(t)​]=[α(t)ω(t)​]

Where:

  • α(t)\alpha(t)α(t): linear acceleration
  • ω(t)\omega(t)ω(t): angular velocity

Continuous-Time Dynamics​

The state equation for the unicycle model is:

x˙(t)=[v(t)cos⁡θ(t)v(t)sin⁡θ(t)00]+[00α(t)ω(t)]=f(x(t),u(t))\dot{x}(t) = \begin{bmatrix} v(t)\cos{\theta(t)} \\ v(t)\sin{\theta(t)} \\ 0 \\ 0 \end{bmatrix} + \begin{bmatrix} 0 \\ 0 \\ \alpha(t) \\ \omega(t) \end{bmatrix} = f(x(t), u(t))x˙(t)=​v(t)cosθ(t)v(t)sinθ(t)00​​+​00α(t)ω(t)​​=f(x(t),u(t))
Nonlinearity

This is a nonlinear system due to the trigonometric functions in the kinematic equations.

Discretization​

For numerical implementation, we discretize the system with sample time TsT_sTs​: x[k+1]=fd(x[k],u[k])x[k+1] = f_d(x[k], u[k])x[k+1]=fd​(x[k],u[k])

where fdf_dfd​ represents the discrete-time dynamics obtained through numerical integration methods.

Problem 1: Parking Problem​

The parking problem represents the most basic optimal control scenario - moving from an initial state to a desired final state.

Problem Setup​

System: Discrete-time unicycle model x[k+1]=fd(x[k],u[k])x[k+1] = f_d(x[k], u[k])x[k+1]=fd​(x[k],u[k])

Initial and target states:

x[0]=[px0py0v0θ0],xd=[pxdpyd00]x[0] = \begin{bmatrix} p_{x0} \\ p_{y0} \\ v_0 \\ \theta_0 \end{bmatrix}, \quad x_d = \begin{bmatrix} p_{xd} \\ p_{yd} \\ 0 \\ 0 \end{bmatrix}x[0]=​px0​py0​v0​θ0​​​,xd​=​pxd​pyd​00​​

Terminal state: At time k=Nk = Nk=N: x[N]=fd[⋯[fd[x[0],u[0]],u[1],…,u[N−1]]]x[N] = f_d[\cdots[f_d[x[0], u[0]], u[1], \ldots, u[N-1]]]x[N]=fd​[⋯[fd​[x[0],u[0]],u[1],…,u[N−1]]]

Key Insight

The final state depends only on the initial state and the sequence of control inputs u[0],u[1],…,u[N−1]u[0], u[1], \ldots, u[N-1]u[0],u[1],…,u[N−1].

Performance Function​

The basic performance function (cost function) is:

J=(x1[N]−xd1)2+(x2[N]−xd2)2+(x3[N]−xd3)2+(x4[N]−xd4)2=(x[N]−xd)T(x[N]−xd)=∥x[N]−xd∥2=e[N]Te[N]=∥e[N]∥2\begin{aligned} J &= (x_1[N] - x_{d1})^2 + (x_2[N] - x_{d2})^2 + (x_3[N] - x_{d3})^2 + (x_4[N] - x_{d4})^2 \\ &= (x[N] - x_d)^T(x[N] - x_d) = \|x[N] - x_d\|^2 \\ &= e[N]^T e[N] = \|e[N]\|^2 \end{aligned}J​=(x1​[N]−xd1​)2+(x2​[N]−xd2​)2+(x3​[N]−xd3​)2+(x4​[N]−xd4​)2=(x[N]−xd​)T(x[N]−xd​)=∥x[N]−xd​∥2=e[N]Te[N]=∥e[N]∥2​

where e[k]=x[k]−xde[k] = x[k] - x_de[k]=x[k]−xd​ is the error vector.

Control Policy​

The optimal control policy seeks: u∗=[u[0],u[1],…,u[N−1]]=arg⁡min⁡u∈ΩJu^* = [u[0], u[1], \ldots, u[N-1]] = \arg\min_{u \in \Omega} Ju∗=[u[0],u[1],…,u[N−1]]=argminu∈Ω​J

where Ω\OmegaΩ is the set of admissible controls.

Weight Matrix​

To prioritize certain states, we introduce a weight matrix: J=e[N]TSe[N]≜∥x[N]−xd∥S2J = e[N]^T S e[N] \triangleq \|x[N] - x_d\|_S^2J=e[N]TSe[N]≜∥x[N]−xd​∥S2​

where SSS is a positive semi-definite symmetric matrix (xTSx≥0x^T S x \geq 0xTSx≥0 and sij=sjis_{ij} = s_{ji}sij​=sji​).

Weight Matrix Properties
  • Usually diagonal: sij=0s_{ij} = 0sij​=0 for i≠ji \neq ji=j
  • Larger diagonal elements indicate higher importance of corresponding states
  • Must be positive semi-definite for convexity

Constraints​

Physical limitations impose constraints:

State constraints: −vmax⁡<v[k]<vmax⁡-v_{\max} < v[k] < v_{\max}−vmax​<v[k]<vmax​

Input constraints: umin⁡<u[k]<umax⁡u_{\min} < u[k] < u_{\max}umin​<u[k]<umax​

Hard Constraints

These are hard constraints that must be strictly satisfied during system operation.

Problem 2: Input Cost Consideration​

Real systems require energy to operate, making input cost an important consideration.

Soft Constraints​

Unlike hard constraints, soft constraints aren't strictly enforced but are penalized in the cost function.

Modified Cost Function​

J=∥x[N]−xd∥S2+∑k=0N−1∥u[k]∥R2J = \|x[N] - x_d\|_S^2 + \sum_{k=0}^{N-1} \|u[k]\|_R^2J=∥x[N]−xd​∥S2​+∑k=0N−1​∥u[k]∥R2​

Where:

  • ∥x[N]−xd∥S2=(x[N]−xd)TS(x[N]−xd)\|x[N] - x_d\|_S^2 = (x[N] - x_d)^T S (x[N] - x_d)∥x[N]−xd​∥S2​=(x[N]−xd​)TS(x[N]−xd​): terminal cost
  • ∥u[k]∥R2=u[k]TRu[k]\|u[k]\|_R^2 = u[k]^T R u[k]∥u[k]∥R2​=u[k]TRu[k]: input cost
  • SSS: reference weight matrix
  • RRR: input cost weight matrix

Design Trade-offs​

The relative weighting determines system behavior:

S≫RS \gg RS≫R: Control performance is more important!

R≫SR \gg SR≫S: Input cost is more important!

Problem 3: Trajectory Tracking​

Moving beyond point-to-point control, trajectory tracking follows a time-varying reference.

Reference Trajectory​

xd[k]=[pxd[k]pyd[k]00]x_d[k] = \begin{bmatrix} p_{xd}[k] \\ p_{yd}[k] \\ 0 \\ 0 \end{bmatrix}xd​[k]=​pxd​[k]pyd​[k]00​​

Performance Function​

J=∥x[N]−xd[N]∥S2+∑k=0N−1(∥x[k]−xd[k]∥Q2+∥u[k]∥R2)J = \|x[N] - x_d[N]\|_S^2 + \sum_{k=0}^{N-1} \left(\|x[k] - x_d[k]\|_Q^2 + \|u[k]\|_R^2\right)J=∥x[N]−xd​[N]∥S2​+∑k=0N−1​(∥x[k]−xd​[k]∥Q2​+∥u[k]∥R2​)

where QQQ is the trajectory weight matrix that penalizes deviations from the reference path at each time step.

Problem 4: Collision Avoidance​

Advanced control problems must consider safety and obstacle avoidance.

Method 1: Hard Constraint Approach​

Add constraint conditions to Problem 3: (px[k],py[k])∉(pxo,pyo)(p_x[k], p_y[k]) \notin (p_{xo}, p_{yo})(px​[k],py​[k])∈/(pxo​,pyo​)

where (pxo,pyo)∈Ωo(p_{xo}, p_{yo}) \in \Omega_o(pxo​,pyo​)∈Ωo​ represents the obstacle space.

Method 2: Soft Constraint Approach​

Modify the cost function to penalize proximity to obstacles:

J=∥x[N]−xd[N]∥S2+∑k=0N−1(∥x[k]−xd[k]∥Q2+∥u[k]∥R2)+∑k=0N−1D−1[k]PJ = \|x[N] - x_d[N]\|_S^2 + \sum_{k=0}^{N-1} \left(\|x[k] - x_d[k]\|_Q^2 + \|u[k]\|_R^2\right) + \sum_{k=0}^{N-1} D^{-1}[k]_PJ=∥x[N]−xd​[N]∥S2​+∑k=0N−1​(∥x[k]−xd​[k]∥Q2​+∥u[k]∥R2​)+∑k=0N−1​D−1[k]P​

Where:

  • D[k]D[k]D[k]: distance from object to obstacle at time kkk
  • PPP: weight matrix for distance penalty
  • D−1[k]PD^{-1}[k]_PD−1[k]P​: penalty that increases as distance decreases

General Optimal Control Problem Formulation​

System Model​

x[k+1]=fd(x[k],u[k],k)x[k+1] = f_d(x[k], u[k], k)x[k+1]=fd​(x[k],u[k],k)

Reference​

xd[k] (may be time-varying)x_d[k] \text{ (may be time-varying)}xd​[k] (may be time-varying)

Performance Function​

J=h(x[N],xd[N],N)+∑k=0N−1g(x[k],xd[k],u[k],k)J = h(x[N], x_d[N], N) + \sum_{k=0}^{N-1} g(x[k], x_d[k], u[k], k)J=h(x[N],xd​[N],N)+∑k=0N−1​g(x[k],xd​[k],u[k],k)

Constraints​

  • State constraints: x[k]∈Xx[k] \in Xx[k]∈X (admissible trajectory set)
  • Input constraints: u[k]∈Ωu[k] \in \Omegau[k]∈Ω (admissible control set)
Feasibility

There must be at least one feasible solution within the constraints. Overly restrictive constraints can lead to infeasible problems.

Common Optimal Control Problem Types​

1. Minimum Time Problem​

J=∑k=0N1=NJ = \sum_{k=0}^N 1 = NJ=∑k=0N​1=N

Objective: Reach the target in minimum time.

2. Terminal Control Problem​

J=∥x[N]−xd[N]∥S2J = \|x[N] - x_d[N]\|_S^2J=∥x[N]−xd​[N]∥S2​

Objective: Minimize final state error.

3. Minimum Input Problem​

J=∑k=0N∥u[k]∥R[k]2J = \sum_{k=0}^N \|u[k]\|_{R[k]}^2J=∑k=0N​∥u[k]∥R[k]2​

Objective: Minimize control effort.

4. Trajectory Tracking Problem​

J=∑k=0N∥x[k]−xd[k]∥Q[k]2J = \sum_{k=0}^N \|x[k] - x_d[k]\|_{Q[k]}^2J=∑k=0N​∥x[k]−xd​[k]∥Q[k]2​

Objective: Follow reference trajectory closely.

5. General Optimal Control Problem​

J=∥x[N]−xd[N]∥S2+∑k=0N−1(∥x[k]−xd[k]∥Q[k]2+∥u[k]∥R[k]2)J = \|x[N] - x_d[N]\|_S^2 + \sum_{k=0}^{N-1} \left(\|x[k] - x_d[k]\|_{Q[k]}^2 + \|u[k]\|_{R[k]}^2\right)J=∥x[N]−xd​[N]∥S2​+∑k=0N−1​(∥x[k]−xd​[k]∥Q[k]2​+∥u[k]∥R[k]2​)

Objective: Balance terminal accuracy, trajectory tracking, and input cost.

6. Regulator Problem​

When xd[k]=0x_d[k] = 0xd​[k]=0, the problem becomes a regulator problem: J=∥x[N]∥S2+∑k=0N−1(∥x[k]∥Q[k]2+∥u[k]∥R[k]2)J = \|x[N]\|_S^2 + \sum_{k=0}^{N-1} \left(\|x[k]\|_{Q[k]}^2 + \|u[k]\|_{R[k]}^2\right)J=∥x[N]∥S2​+∑k=0N−1​(∥x[k]∥Q[k]2​+∥u[k]∥R[k]2​)

Problem Transformation

For trajectory problems, introduce error e[k]=x[k]−xd[k]e[k] = x[k] - x_d[k]e[k]=x[k]−xd​[k] and formulate the error dynamics. The control task becomes driving e[k]→0e[k] \to 0e[k]→0, which can be solved as a regulator problem.

Solution Approaches​

Different optimal control problems require different solution methods:

  1. Linear Quadratic Regulator (LQR): For linear systems with quadratic costs
  2. Dynamic Programming: For general nonlinear problems
  3. Model Predictive Control (MPC): For real-time implementation with constraints
  4. Pontryagin's Maximum Principle: For continuous-time problems
  5. Direct Methods: Numerical optimization of discretized problems
